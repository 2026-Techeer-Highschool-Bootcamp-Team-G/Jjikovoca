# 백엔드 미완 API 개발 순서 로드맵

auth 도메인(회원가입·로그인·재발급·로그아웃 + JWT 인증 필터)이 완성됐다. Notion API 명세(45개) 중
**보류(발음설정 재정의·결제 PG 21~24)를 뺀 미완 API 35개**를 아래 순서로 개발한다.

## 도메인 의존 순서
```
auth(✅완료) → core.card → analysis(Gemini) → core.review(학습) → core.stats(리포트·게이미)
```
- `core.card` ← auth (인증)
- `analysis` ← core.card (`AnalyzeRequested` 이벤트 수신, `analyze_job` 소유)
- `core.review` ← core.card (card 엔티티)
- `core.stats` ← core.review (study_log 집계)

인프라(V1 14테이블·shared·app/config·Redis·Swagger)는 이미 구현됨. core/analysis 패키지는 스텁(package-info)만 있다.

## 진행 방식
- 백엔드 워크트리 `Jjikovoca-server/`(sparse `server/`)에서만 작업.
- 각 기능(또는 응집된 소묶음)마다 **이슈 → 브랜치(`gh issue develop`) → 파일 단위 커밋 → PR(`Closes #N`)**.
- 설계 원칙(문서 13): 엔티티 패키지 비공개·노출은 DTO로만 / 도메인 간 통신은 shared 이벤트 /
  quota·판정은 원자적 트랜잭션 / study_log·exp_log는 INSERT-only / **정답(answer_value·해설·단계 내용)
  조회 응답 미노출** / Redis 캐시-어사이드 / FSRS는 `fsrs_state IS NULL` 단일 분기.

## 제외·정정 (개발 대상 아님)
- **API-4 발음설정(PATCH /me/settings)** — 개발 제외. 발음은 **영어 단어 카드에서 클릭 시 미국식/영국식
  재생**이며 프론트 Web Speech가 `card.word`로 처리한다. 백엔드 API·컬럼 불필요. (Notion "보류"로 갱신)
- **API-38 로그아웃** — Notion "개발전"이나 **실제 완료(PR #20)**. (Notion "완료"로 갱신)

## Phase 0 — 사용자·시스템 워밍업 (인증 필터 첫 실검증)
| API | 엔드포인트 | 내용 |
|---|---|---|

## Phase 1 — core.card (카드 도메인)
| API | 엔드포인트 | 내용 |
|---|---|---|
| 6 캡처 분석(접수) | `POST /api/cards/analyze` | **quota 원자적 차감** → analyze_job 생성 → `AnalyzeRequested` 발행 → 202+jobId |
| 7 카드 피드 | `GET /api/cards` | 과목 탭·정렬·페이지. 정답 미노출 |
| 8 카드 상세 | `GET /api/cards/{id}` | 소유자 검증·힌트 게이팅 |
| 9 카드 삭제 | `DELETE /api/cards/{id}` | soft delete |
| 10 크롭 이미지 | `GET /images/{file}` | 인증 예외 경로, 이미지 서빙 |
| 36 원문 보관함 | `GET /api/cards/archive` | 일자별 크롭 열람(image_path 재사용) |

신규: `Card`(STI WORD/PROBLEM)·`UserQuotaDaily` 엔티티, `CardService`·`QuotaService`(조건부 UPDATE), DTO(answerValue 제외).

## Phase 2 — analysis (모의 Gemini)
카드 생성 흐름을 **모의 모드**로 완성(실 Gemini는 나중에 `GeminiClient` 구현만 교체).
| API | 엔드포인트 | 내용 |
|---|---|---|
| 6 캡처 분석(처리) | (비동기) | `AnalyzeRequested` 수신 → 모의 응답으로 카드 채움 → `AnalyzeCompleted`/`AnalyzeFailed`(quota 환불) |
| 39 분석 폴링 | `GET /api/cards/analyze/{jobId}` | analyze_job 상태 |
| 40 분석 SSE | `GET /api/cards/analyze/{jobId}/events` | 진행 단계 스트림 |

신규: `AnalyzeJob`(상태머신), `GeminiClient` 인터페이스+**모의 구현**, `AnalysisService`(@Async), Redis `analysis` 캐시.
→ 이 Phase 끝나면 **캡처→카드 생성 E2E**가 모의로 동작.

## 검증 (각 기능마다)
- `./gradlew build` — 컴파일 + **ArchUnit 경계 테스트** 통과.
- `bootRun` 후 **Swagger**(`/swagger-ui.html`)에서 수동 시험(Authorize에 JWT).
- 핵심 E2E: 로그인 → analyze(모의) → 카드 생성 → 피드/상세 → 학습 기록 → 리포트.
- 통합 테스트: Testcontainers(실 MySQL/Redis). Notion 개발상태 단계별 갱신.
